# 📘 Руководство к файлу `game_of_thrones_deaths+adapt.py`

## О чём этот проект

Данный скрипт был создан в рамках домашнего задания **Школы глубокого обучения МФТИ**.  
Его задача — предсказать, какие персонажи вселенной «Игры престолов» останутся в живых, а какие погибнут, используя методы машинного обучения и анализа данных.

В основе лежит **логистическая регрессия** — классический алгоритм классификации, который хорошо подходит для бинарных прогнозов (жив / мёртв).

---

## Исходные данные

Программа работает с тремя файлами, которые автоматически подгружаются перед началом работы:

1. **Обучающий набор** – данные о персонажах с известным ответом (жив ли персонаж).
2. **Тестовый набор** – те же признаки, но без целевой переменной. Именно для этих персонажей нужно сделать предсказание.
3. **Пример файла ответа** – шаблон, в который будут записаны результаты.

Все файлы хранятся в открытом доступе и загружаются по прямым ссылкам.

---

## Признаки персонажей

Для каждого героя в датасете собрано множество характеристик:

| Категория | Примеры признаков |
|-----------|-------------------|
| 👤 Личные данные | имя, возраст, пол, дата рождения |
| 👑 Социальный статус | титул, знатность, дом, культурная группа |
| 📚 Появление в книгах | флаги участия в каждой из пяти книг |
| 💍 Семейное положение | супруг(а), жив ли супруг, женат/замужем ли |
| 👪 Родственники | отец, мать, наследник, живы ли они |
| ⚰️ Смерти в семье | количество умерших родственников |
| 🌐 Популярность | число внутренних ссылок на страницу персонажа в вики |

**Целевая переменная** `isAlive` принимает два значения:
- `1` – персонаж жив к моменту событий книг,
- `0` – персонаж погиб.

## Этапы обработки данных

Чтобы превратить сырые данные в пригодные для обучения модели, выполняются следующие шаги.

### 1️⃣ Очистка и заполнение пропусков

- Часть записей не содержит информации о возрасте, культуре или доме. Такие пропуски заполняются нулевыми значениями или специальными индикаторами.
- Некоторые столбцы (например, дата рождения) признаны неинформативными и удаляются.

### 2️⃣ Создание новых признаков

Из исходных данных выводятся дополнительные характеристики:
- **Логарифмическая популярность** – исходное значение популярности сильно искажено (есть очень популярные и почти неизвестные персонажи), логарифмирование делает распределение более компактным.
- **Индикатор наличия умерших родственников** – бинарный признак, который показывает, есть ли у персонажа хотя бы один мёртвый родственник.
- **Группы культур** – множество мелких культур объединяются в крупные категории (например, «народ Севера», «жителе Железных островов», «эссосские нации»). Это позволяет модели лучше обобщать.
- **Признаки отсутствия данных** – создаются отдельные колонки, которые указывают, было ли значение возраста или культуры неизвестно изначально.

### 3️⃣ Преобразование категориальных признаков

Категории в столбцах `title`, `house` и `culture_grouped` преобразуются с помощью **one-hot кодирования**.  
Каждое уникальное значение становится отдельным бинарным столбцом со значениями 0 или 1.

### 4️⃣ Разделение данных

Обучающая выборка случайным образом делится на две части:
- **80%** – для тренировки модели,
- **20%** – для проверки качества (модель их не видит во время обучения).

Чтобы результаты были воспроизводимыми, используется фиксированное случайное зерно.

---

## Обучение модели

Алгоритм **логистической регрессии** обучается на подготовленных данных.  
Он пытается найти линейную зависимость между входными признаками и вероятностью того, что персонаж жив.

После обучения модель делает предсказания на **тестовом наборе** (на самом деле — на отложенной валидационной выборке) и вычисляет долю правильных ответов.

---

## 📊 Результаты качества

На валидационной выборке (20% от обучающих данных) модель достигает **точности (accuracy) 0.83**, то есть **83% правильных предсказаний**.  

Это означает, что из 100 персонажей, которых модель не видела во время обучения, она верно определяет судьбу 83 героев. Такой результат считается очень хорошим для задачи с неполными и шумными данными.

Итоговая модель, обученная на всех доступных тренировочных данных, затем применяется к тестовому набору для формирования финального ответа.

## Особенности и примечания

- Проект создавался в образовательных целях, поэтому некоторые решения (например, группировка культур) основаны на знании вселенной «Игры престолов». При использовании на других данных эти правила могут потребовать корректировки.
- Логистическая регрессия – это базовая модель, которая даёт точность **0.83**. Для улучшения качества можно попробовать градиентный бустинг или случайный лес.
- Точность работы на валидационной выборке может незначительно меняться при разных разбиениях, но из‑за фиксированного `random_state` все запуски дают одинаковый результат.
